# BAMOE Developer Tools Extension - PatternFly TSX Forms

This directory contains TypeScript .tsx forms generated by the BAMOE Developer Tools VS Code extension using PatternFly library.

## Extension Workflow (IBM BAMOE 9.3.x Documentation)

1. **Install Extension**: BAMOE Developer Tools in VS Code (v9.3.x compatible)
2. **Open BPMN**: Right-click on user task in BPMN diagram
3. **Design Form**: Select "Design Form" > "PatternFly Form" > "TypeScript"
4. **Configure Fields**: Add input fields with PatternFly 5.x components
5. **Generate**: Extension creates .tsx files and HTML wrappers
6. **Build**: Extension compiles TypeScript to JavaScript automatically

## Generated Files

### TypeScript Components (.tsx)
- `src/main/typescript/forms/DcssChildrenForm.tsx` - Children count form component
- `src/main/typescript/forms/DcssDependentForm.tsx` - Dependent information form component

### Compiled JavaScript (.js)
- `src/main/resources/META-INF/resources/js/dcss-children-form.js`
- `src/main/resources/META-INF/resources/js/dcss-dependent-form.js`

### HTML Wrappers
- `dcss-children-form.html` - Loads children form component
- `dcss-dependent-form.html` - Loads dependent form component

## Build Process

The extension automatically:
1. Compiles .tsx files to JavaScript
2. Generates HTML wrapper files
3. Sets up CDN dependencies
4. Configures webpack for optimal bundling

## Development

Forms are built with:
- **TypeScript 5.3+** for type safety
- **React 18.3+** with hooks
- **PatternFly 5.1+** for UI components
- **Webpack 5.89+** for bundling
- **BAMOE 9.3.x Process Integration** for workflow

## Process Integration

Forms submit data to BAMOE process variables:
- `childrenCount` - Number of children
- `parentA_GrossIncome` - Parent A income
- `parentB_GrossIncome` - Parent B income
- `custodyPercentageHigh` - Custody percentage
- `dependentInfo` - Dependent details array

Results are calculated using California Family Code §4055 formula.
